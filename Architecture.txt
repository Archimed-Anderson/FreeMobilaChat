Notre solution consiste à capter les tweets concernant Free sur Twitter/X, puis à les analyser automatiquement avec un modèle de classification fine-tuné pour détecter ceux qui sont de vraies réclamations. Lorsqu’un tweet est identifié comme une réclamation, un message de réponse personnalisé est généré et publié en commentaire, contenant un lien vers un chatbot. Ce chatbot demande d’abord trois informations obligatoires (Nom, Prénom, Problème) et, dès que l’utilisateur les fournit, un ticket est créé automatiquement dans une interface interne pour assurer le suivi. En parallèle, le chatbot tente d’aider le client grâce à un modèle de langage connecté à une base de connaissances (FAQ Free, Assistant Free, procédures internes). Si le problème est résolu par le bot, le ticket est clos automatiquement ; sinon, après plusieurs tentatives infructueuses, le ticket est transmis à un agent humain qui reprend la main. Enfin, l’interface interne centralise les tickets avec leur historique et des KPIs (taux de résolution automatique, délais, taux d’escalade…) afin de mesurer et améliorer la qualité du service.
